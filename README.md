# ATROPOS

**The Sovereign Local-First Deterministic Hierarchical Multi-Swarm Software Development Engine**

![Hero](docs/assets/imported/20260629_190733/image-14.jpg)

**Independent • Runs for days without stopping • No quota collapse • Preventive verification • E(Δ)=0**

ATROPOS is not another coding agent. It is a full autonomous software-development engine built on a strict hierarchical territory model. This lets it work for days on end with no quota problems, no uncontrolled information leaks, and real preventive verification.

Current systems (Codex CLI, Claude Code, Antigravity, OpenSwarm, etc.) all rely on chatty LLM coordination. This causes exploding token costs, reactive drift detection, and makes long-running work unsustainable.

ATROPOS replaces that with explicit territory assignment, Director-level preventive diff monitoring, a controlled HR Router, and deterministic verification before any LLM escalation.

---

![Hierarchy](docs/assets/imported/20260629_190733/image-19.jpg)

**Core Hierarchy**
- Human Owner/CEO: Strategic direction and final authority
- Director: Territory assignment + global diff monitoring + preventive drift detection
- VP / Manager: Domain and team coordination
- Specialist / Worker: Bounded execution inside assigned territory only
- Custodian + Auditor: Hygiene and independent verification
- HR Router: Single controlled channel for any cross-boundary information

Most work stays inside its territory and never needs heavy LLM coordination.

**Key Advantages**
- Coordination cost scales linearly with active territories (not agent count)
- Drift detection is preventive and cheap
- Global visibility is concentrated in the Director
- Information flow is controlled and auditable
- Can run for days without quota or context collapse
- Strong nano-style coherent batch discipline with E(Δ)=0 gates

**Provider System**  
Free-first by default with 30+ providers and full fixture coverage. Paid providers require explicit unlock. QuotaLedger prevents collapse during long runs.

**Current State**  
Reactive Termux TUI with persistent tabs and responsive layout. Complete provider routing, redaction, and verification foundation. All development follows the 20-phase blueprint.

**Next (Pass 5)**: Provider activation doctor, truthful key/provider state, and interactive route inspection.
